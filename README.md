# 分布式即时通讯系统

## 项目简介
一个从架构设计到编码落地的分布式即时通讯系统，支持用户注册登录、
好友管理与点对点实时聊天。系统拆分网关、状态、聊天等多个独立服务，
服务间通过 gRPC 通信，数据层使用 MySQL 与 Redis。

## 技术栈
C++、Boost.Asio、gRPC、Protobuf、MySQL、Redis、CMake、Qt

## 架构说明
• GateServer：接收客户端 HTTP POST 请求，解析路由后通过 gRPC 分别调用
  StatusServer 与 ChatServer 处理业务，处理完毕后断开连接（短连接）
• StatusServer：管理用户在线状态与所在聊天节点信息，数据存入 Redis
• ChatServer：与客户端维持 TCP 长连接，负责点对点实时消息转发，
  同时通过 gRPC 与 StatusServer 通信查询用户状态
• 验证码功能：在VarifyServer内直接生成校验码，存入 Redis 并设置有效期，
  调用第三方邮件服务完成发送

## 已实现功能
• 用户注册、登录、密码重置（含邮件验证码）
• 好友搜索、好友申请发送与处理
• 用户信息与好友关系持久化落盘（MySQL）
• 点对点实时聊天消息转发（跨节点支持）
• 在线状态管理（Redis 缓存当前在线用户与所在服务器）

## 未实现 / 待优化
• 聊天消息未落盘，退出即消失（已规划消息表结构，后续可扩展）
• 当前为学习型项目，未进行大规模压测与生产级容错

## 编译说明
项目涉及多个服务模块，依赖 Boost、gRPC、Protobuf、MySQL、Redis 等库，
环境配置较为复杂。建议先通过源码了解架构设计即可。

## 关于代码
• 服务间 gRPC 通信由框架内置自动重连，业务层无需额外处理

gRPC 通信、缓存策略及工程落地的理解，为开发更复杂的服务端系统打下基础。
